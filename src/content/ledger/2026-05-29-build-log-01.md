Epic 17 was one architectural change carried all the way through: make the agent a pure
advisor that can propose a move but can never sign one. Before this, the hot key and the
whole signing path lived inside the agent — the worst possible place for it. The component
making autonomous decisions was also the component holding the ability to broadcast them.

So we moved the key. The `TransactionSigner`, the `NonceManager`, the gas economics, and the
policy-guarded signing orchestration all relocate into the overseer. What is left in the agent
is the ability to emit a structured intent — *deposit this much of this asset into that
protocol* — and then block, waiting on the overseer through approval, execution, and receipt.

The design we locked is what the epic calls Model B. The overseer does not trust calldata
handed to it by the agent. It rebuilds the calldata itself, from the persisted structured
intent. The agent's vocabulary — the set of intents it is allowed to express — *is* the
policy. The alternative, decoding agent-supplied calldata and checking it after the fact, was
rejected as too fragile to lean on.

Approval runs in two tiers. A set of trusted policy gates — the kill-switch, the whitelist,
the supply cap, the gas ceiling, and a new per-transaction and daily value cap — run on every
signed transaction. "Auto-approve," when it is on, only ever skips the *human* tier; it never
skips the policy tier. An over-cap intent is forced to a human or blocked outright, no matter
how auto-approve is set.

To let both sides share the on-chain plumbing without the agent inheriting any signing power,
the protocol adapters and read sources moved into a new domain package called `chain`. It
depends on viem but holds no wallet client. Both the agent and the overseer import it — the
agent for reads, the overseer for reads plus calldata-building plus signing. Two boundary
invariants now hold and are checked in the boundaries test: the agent has zero signing
capability, and the chain package holds no wallet client. The NFR12 rule — that a wallet
client may be created in only one place — relocates from the agent to the overseer. NFR13,
that the agent never imports the overseer, is unchanged.

The sequencing was deliberate. The operator's call on 2026-05-29 was to land this before any
mainnet transaction, precisely because building agent-side stopgaps first would mean deleting
them again at Story 17.6, when the agent's signer comes out for good. By the end of that
story the agent has no key, no signer, and no signing code at all. It proposes, and it waits.
